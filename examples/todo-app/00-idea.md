# Idea

> A dead-simple shared todo list for a household — add things, check them off, everyone sees the same list.

(Stage 1 / `/office-hours` would interrogate this: who's it for, riskiest assumption, smallest valuable build. For this example we take the go decision as given and start the ladder at Stage 2.)

**Office-hours takeaways (abbreviated):**
- User: members of one household sharing a single list.
- Riskiest assumption: people will keep it in sync if it's frictionless — so *real-time shared state* is the wedge, not features.
- Smallest valuable build: one shared list, add + complete, live across devices. No accounts beyond a shared household.
