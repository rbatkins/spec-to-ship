# Worked example — a todo app

One idea walked through the whole spec-to-ship loop, so you can see the *shape* of each artifact before you trust the process on your own idea.

| File | Stage |
|------|-------|
| [`00-idea.md`](00-idea.md) | The one-line input |
| [`10-roadmap.md`](10-roadmap.md) | Stage 2 — sequenced outcomes |
| [`20-epics.md`](20-epics.md) | Stage 3 — epics |
| [`30-stories.md`](30-stories.md) | Stage 4 — one-PR stories with acceptance criteria |
| [`40-sprint.md`](40-sprint.md) | Stage 5 — the first sprint, ready for the execution loop |

*(Stage 1 / office-hours output and Stage 6 PRs are not reproduced here — Stage 1 is a gstack design doc, and Stage 6 produces real PRs in your codebase.)*

This example is deliberately tiny. The point is the **ladder**, not the product.
