# Stage 1 — First principles

**Goal:** decide whether the idea is worth building and define what it actually is — *before* any backlog exists.

**Powered by:** [gstack](https://github.com/garrytan/gstack) `/office-hours`.

## Run

1. Take the one-line idea exactly as given.
2. Run gstack **`/office-hours`** on it. Let it interrogate the idea: who is it for, what's the sharpest version of the problem, what's the riskiest assumption, what's the smallest thing worth building first, and is it worth building at all.
3. Capture the output as a short **design doc** + an explicit **go / no-go**.

## Output (hand to Stage 2)

- `design-doc.md` — problem, target user, the wedge, riskiest assumptions, the smallest valuable first build.
- A one-line **go / no-go**. If no-go, **stop here** — that's a success, not a failure.

## Guardrails

- Do **not** produce a roadmap or tasks yet. This stage only decides *what* and *whether*.
- If the idea is too vague to office-hours, ask the human 2–3 sharpening questions first.
- Prefer killing or narrowing a weak idea now over decomposing it later.
